## [1.2.0] - 2026-04-19

### Fixed
- **合计翻倍 bug**：合计行原用 `SUMIF(H:H,">0")` 会将小计行金额重复计入，导致总金额翻倍。改用 `SUMPRODUCT((ISNUMBER(A列))*(H列))`，仅统计序号列为整数的数据行，彻底排除小计行干扰。
- **追加后数据丢失 bug**：`delete_rows` 删除行内容但不清理已注册的合并范围，导致旧合并范围残留，下次读取时被覆盖区域的单元格返回 `None`，追加时所有字段变为"待补"。修复方案：在 `delete_rows` 前先对数据区域（`row >= DATA_START`）执行 `unmerge_cells`，彻底清除残留合并。

### Changed
- 列顺序调整：金额（元）列从第 4 列移至第 8 列（备注列左侧），使描述性字段连续排列
- 小计行序号列留空，不再与费用类型列合并，保持序号视觉连续性
- 所有金额单元格加粗显示
- 所有单元格内容改为居中对齐

## [1.1.0] - 2026-04-18

### Added
- **使用前向导**：首次运行询问报账名称，作为文件名和 Excel 标题
- **跨会话追加**：文件已存在时直接追加，不覆盖历史数据，读取时自动跳过小计/合计行
- **完整分类体系**：覆盖高校/科研机构报账系统全部类目
  - 报账系统 21 类（办公用品、市内交通费、国内差旅费等）
  - 酬金系统 15 类（课酬、编审费、科研劳务费等），全部自动标记"无发票"
- **费用类型列前置**：移至序号列右侧，同类型多行自动合并单元格
- **分类小计行**：每个费用类型下方插入绿色小计行（SUM 公式）
- **合计行**：表格底部金色合计行

### Changed
- Excel 明细 Sheet 列顺序：序号 → 费用类型 → 发生日期 → 金额 → 事由/摘要 → 付款方式 → 发票状态 → 经办人 → 备注 → 录入时间
- README 演示表格改用 HTML `<table>` + `rowspan` 实现真实合并单元格效果

## [1.0.0] - 2026-04-18

### Added
- 初始发布
- 自然语言输入，批量解析（分号分隔）
- 智能推断费用类型、发票状态、付款方式
- 双 Sheet 输出：明细表 + 汇总表（SUMIF 公式动态更新）
- 交替行色、冻结表头、金额格式化
- 跨平台兼容（Windows / macOS / Linux），默认输出目录 `~/expense-reports/`
- CLI 接口：`--file`、`--title`、`--data`（JSON）
