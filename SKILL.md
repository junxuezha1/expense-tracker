# expense-tracker — 报账账目整理

## 概述

接受自然语言描述的费用条目，解析结构化字段，追加到 Excel 明细表，并自动维护汇总 Sheet。

**依赖**：Python 3.8+，`openpyxl`（`pip install openpyxl`）

**脚本路径**：`excel_ops.py` 与本 SKILL.md 在同一目录。调用时动态构建路径：
- 全局安装：`~/.claude/skills/expense-tracker/excel_ops.py`
- 项目安装：`.claude/skills/expense-tracker/excel_ops.py`
- 用 Python 的 `Path.home()` 展开 `~`，确保跨平台兼容

**输出目录**：默认 `~/expense-reports/`（跨平台），用户可在向导中指定其他路径。

---

## 工作流

### Step 0：使用前向导（每次新建文件时执行）

**首次启动时**，先询问用户：

> 请问本次报账的名称是什么？（例如：2026年4月差旅费、项目X劳务报销）
> 这将作为 Excel 文件名和表格标题。
>
> 输出目录（直接回车使用默认 `~/expense-reports/`，或输入自定义路径）：

- 文件路径：`{输出目录}/{名称}.xlsx`
- 如果该文件已存在，跳过向导，直接进入 Step 1（追加模式）
- 如果用户未指定名称，默认使用当前年月，如 `2026年4月报账`

---

### Step 1：解析输入

从用户的自然语言中提取以下字段：

| 字段 | 说明 | 缺失时 |
|------|------|--------|
| 发生日期 | 费用发生日期，如"昨天"→推算具体日期 | 标记"待补" |
| 金额（元） | 数字，去掉"元"/"块" | 必须有，否则询问 |
| 费用类型 | 见下方分类表（报账系统/酬金系统） | 根据事由推断，不确定则询问 |
| 事由/摘要 | 简短描述这笔钱的用途 | 标记"待补" |
| 付款方式 | 对公 / 对私 | 标记"待补" |
| 发票状态 | 有发票 / 无发票 / 待补 | 见下方规则 |
| 经办人 | 姓名（选填） | 留空 |
| 备注 | 补充说明（选填） | 留空 |

**发票状态规则：**
- 酬金系统所有类型 → 发票状态自动=无发票（不询问）
- "没发票/无票/不含票" → 发票状态=无发票
- "有发票/含票/附发票" → 发票状态=有发票
- 其余报账系统类型 → 标记"待补"

**付款方式规则：**
- "对公/公司卡/公务卡/对公账户" → 付款方式=对公
- "自己付/垫付/私人/对私/个人" → 付款方式=对私

### Step 2：展示解析结果并确认

以表格形式展示解析出的所有条目，标注"待补"字段，询问用户是否有需要修改或补充的内容（可跳过）。

**展示格式**（列顺序与 Excel 一致）：

| # | 费用类型 | 发生日期 | 金额 | 事由 | 付款方式 | 发票状态 | 经办人 | 备注 |
|---|----------|----------|------|------|----------|----------|--------|------|
| 1 | 市内交通费 | 2026-04-17 | 50 | 打车 | 对私 | 无发票 | | |
| 2 | 公务招待费 | 2026-04-18 | 320 | 招待客户午饭 | 对公 | 有发票 | | |
| 3 | 课酬 | 2026-04-18 | 800 | 张老师授课 | 对私 | **无发票（自动）** | | |

底部附：总金额 ¥X,XXX.00，共 N 条。

---

### Step 3：写入 Excel

确认后调用 `excel_ops.py`（路径根据实际安装位置动态构建）：

```bash
python "/path/to/skills/expense-tracker/excel_ops.py" \
  --file "/path/to/output/{名称}.xlsx" \
  --title "{名称}" \
  --data '<JSON字符串>'
```

JSON 格式：
```json
[
  {
    "发生日期": "2026-04-18",
    "金额": 50.0,
    "费用类型": "交通",
    "事由": "打车去客户处开会",
    "付款方式": "对私",
    "发票状态": "无发票",
    "经办人": "",
    "备注": ""
  }
]
```

---

### Step 4：报告结果

脚本返回 JSON，向用户报告：
- 新增条数
- 当前总条数
- 文件路径

## Excel 结构

- **明细 Sheet**：标题行（报账名称）+ 表头行 + 数据行（交替底色），金额列右对齐数字格式
- **汇总 Sheet**：总金额（金色高亮）、按付款方式/发票状态/费用类型分类汇总，全部使用 SUMIF 公式动态更新

## 常见问题

- **文件不存在**：自动触发向导，创建新文件
- **文件已存在**：跳过向导，直接追加
- **日期模糊**（"上周五"）：根据当前日期推算，展示给用户确认
- **金额含范围**（"大概100左右"）：询问用户确认具体金额
- **路径含中文/空格**：用引号包裹路径，Python `Path` 会正确处理
